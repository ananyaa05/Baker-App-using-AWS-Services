## AWS Services & Why Each Is Used

| Service | Where | Why |
|---------|-------|-----|
| **VPC** | Network layer | Isolated private network for all resources |
| **Internet Gateway** | VPC | Allows public internet traffic into the VPC |
| **Public Subnets (x2)** | us-east-1a/1b | Hosts ALB — must be publicly reachable |
| **Private Subnets (x2)** | us-east-1a/1b | Hosts EC2 app servers — not public |
| **DB Subnets (x2)** | us-east-1a/1b | Hosts RDS — fully isolated |
| **NAT Gateway** | Public Subnet | Lets EC2 in private subnets reach internet |
| **Route Tables** | VPC | Public → IGW, Private → NAT |
| **ALB-SG** | Security | Allows port 80/443 from internet only |
| **App-SG** | Security | Allows port 80 from ALB-SG only |
| **DB-SG** | Security | Allows port 3306 from App-SG only |
| **IAM EC2 Role** | IAM | EC2 can access S3 + Rekognition |
| **IAM Lambda Role** | IAM | Lambda can access S3 + RDS + SNS + Rekognition |
| **ALB** | Public Subnet | Distributes traffic across EC2 instances |
| **Auto Scaling Group** | Private Subnet | Min 2, Max 5 EC2s · scales on CPU > 70% |
| **EC2 (t3.micro)** | Private Subnet | Runs Flask backend |
| **RDS MySQL** | DB Subnet | Stores detection history |
| **S3** | AWS Managed | Stores photos + results + hosts frontend |
| **Lambda** | AWS Managed | Triggered by S3 upload → calls Rekognition |
| **Rekognition** | AWS Managed | AI image label detection |
| **SNS** | AWS Managed | SMS alert if burnt item detected |

## How It All Works Together

```
Internet
   ↓
Internet Gateway (VPC entry point)
   ↓
ALB — public subnets (load balances traffic)
   ↓
EC2 instances — private subnets (Flask app, Auto Scaling Group)
   ↓
S3 ← EC2 uploads photo here
   ↓ (ObjectCreated trigger)
Lambda — auto fires
   ↓
Rekognition — identifies the pastry
   ↓
RDS — Lambda saves detection result
   ↓
SNS — SMS if burnt detected
   ↓
Result JSON saved back to S3
   ↓
EC2 reads result → returns to frontend → User sees recipe
```

## Every Session — Quick Start

> ⚠️ Sandbox resets every 3 hours. Run this at the start of every session.

### Step 1 — Person 3: Run deploy script in CloudShell
```bash
# Open AWS CloudShell (icon in top navigation bar)
git clone https://github.com/YOUR_USERNAME/baker-app.git
cd baker-app
chmod +x setup/deploy.sh setup/cleanup.sh

# Edit YOUR_PHONE in deploy.sh first!
nano setup/deploy.sh   # change +91XXXXXXXXXX

./setup/deploy.sh
```
Runtime: ~8-10 minutes. Prints all URLs/ARNs at the end.

### Step 2 — Share session-config.txt with team
```
setup/session-config.txt  ← auto-generated by deploy.sh
```
Contains: BUCKET_NAME, ALB_DNS, WEBSITE_URL, SNS_TOPIC_ARN, DB_HOST

### Step 3 — Test it
```bash
# Upload a test image
aws s3 cp cake.jpg s3://YOUR-BUCKET/uploads/cake.jpg

# Watch Lambda logs in real time
aws logs tail /aws/lambda/baker-app-handler --follow

# Check EC2 health
curl http://YOUR-ALB-DNS/health
```

### Before session ends — IMPORTANT
```bash
./setup/cleanup.sh   # deletes NAT GW, RDS, ALB, ASG to save credits
```
NAT Gateway costs ~$0.045/hr. Always clean up!

## Troubleshooting

| Problem | Fix |
|---------|-----|
| Lambda not triggering | Check S3 notification: `aws s3api get-bucket-notification-configuration --bucket BUCKET` |
| Rekognition access denied | Re-run deploy.sh (IAM policy may have been reset) |
| ALB returns 502 | EC2 still booting — wait 2-3 mins after deploy |
| RDS connection refused | Check DB_HOST env var in Lambda: `aws lambda get-function-configuration --function-name baker-app-handler` |
| SNS not sending SMS | Verify phone in SNS > Subscriptions (must be Confirmed status) |
| Frontend shows demo mode | ALB not ready yet or deploy.sh still running |
| Credits running low | Run cleanup.sh immediately — NAT GW is the expensive part |
